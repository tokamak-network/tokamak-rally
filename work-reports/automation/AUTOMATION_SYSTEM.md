# Tokamak Work Automation System
## 목표
매주 Tokamak Network 생태계에 기여하는 프로젝트를 만들고, 기존 프로젝트를 업데이트하며, 이 과정을 체계적으로 자동화한다.

---

## 시스템 구조

```
┌─────────────────────────────────────────────────┐
│                   Jaden (Human)                  │
│            Telegram ↔ 최종 의사결정               │
└──────────────────────┬──────────────────────────┘
                       │
          ┌────────────▼────────────┐
          │   JadenJunior (Main)    │
          │   중간 리더 / 허브 역할    │
          │   - 주간 플래닝           │
          │   - 서브에이전트 관리      │
          │   - 결과 종합 & 보고      │
          └─────┬──────┬──────┬─────┘
                │      │      │
       ┌────────▼──┐ ┌─▼────┐ ┌▼─────────┐
       │ 🔨 Coder  │ │ 🔍 R │ │ 📋 Ops   │
       │ 코드 작업  │ │ 리서치│ │ 업무정리  │
       │ (spawn)   │ │(spawn)│ │ (spawn)  │
       └───────────┘ └──────┘ └──────────┘
```

### 역할 정의

| 역할 | 담당 | 실행 방식 |
|------|------|----------|
| **Main (JadenJunior)** | 중간 리더. 주간 계획 수립, 서브에이전트 배정, 진행상황 종합, Jaden에게 보고 | 메인 세션 (항상 활성) |
| **Coder** | 코드 작성, PR, 버그 수정, 새 프로젝트 scaffold | `sessions_spawn` 서브에이전트 |
| **Researcher** | 생태계 리서치, 새 아이디어 탐색, 경쟁사/트렌드 분석 | `sessions_spawn` 서브에이전트 |
| **Ops** | 동료 업무 팔로업, 문서 정리, 회의 준비, 일정 관리 | `sessions_spawn` 서브에이전트 |

## 구축 단계 (Phase)

### Phase 1: 기반 구축 (Week 1)
> 목표: 주간 사이클이 돌아갈 수 있는 최소 구조 만들기

- [ ] **주간 플래닝 템플릿** 생성 (`memory/weekly/YYYY-WXX.md`)
  - 이번 주 목표, 진행 중 작업, 블로커, 다음 주 예고
- [ ] **프로젝트 레지스트리** 생성 (`PROJECTS.md`)
  - 모든 활성 프로젝트 목록 + 상태 + 담당 + 마지막 업데이트
- [ ] **Cron 스케줄 설정**
  - 매주 월요일 09:00 — 주간 플래닝 세션 (Main이 자체 실행)
  - 매주 금요일 17:00 — 주간 리뷰 & Jaden에게 요약 보고
  - 매일 10:00 — 서브에이전트 상태 체크 & 경량 진행 보고
- [ ] **HEARTBEAT.md 업데이트** — 일일 체크 항목 추가

### Phase 2: 서브에이전트 워크플로우 (Week 2)
> 목표: 서브에이전트가 독립적으로 작업하고 결과를 보고하는 파이프라인

- [ ] **작업 큐 시스템** (`memory/tasks/`)
  - `backlog.md` — 아이디어 & 미배정 작업
  - `in-progress.md` — 현재 서브에이전트가 수행 중
  - `done.md` — 완료된 작업 아카이브
- [ ] **서브에이전트 실행 패턴 정립**
  - Coder: `sessions_spawn(task="...", label="coder-xxx")`
  - Researcher: `sessions_spawn(task="...", label="research-xxx")`
  - 결과물은 워크스페이스 파일로 저장 → Main이 리뷰
- [ ] **결과 보고 자동화**
  - 서브에이전트 완료 시 자동 announce → Main이 종합
  - Main이 Jaden에게 Telegram으로 요약 전달

### Phase 3: 리서치 & 아이디어 파이프라인 (Week 3)
> 목표: 새 프로젝트 아이디어가 자동으로 발굴되는 구조

- [ ] **생태계 모니터링**
  - Tokamak GitHub org 활동 추적
  - 이더리움/L2 생태계 뉴스 모니터링 (web_search 기반)
  - 경쟁 L2 프로젝트 동향 체크
- [ ] **아이디어 보드** (`memory/ideas.md`)
  - Researcher가 발굴한 아이디어 → 주간 플래닝에서 평가
  - Jaden 승인 시 → backlog → 서브에이전트 배정
- [ ] **동료 업무 팔로업 시스템**
  - GitHub org 활동 기반 동료 작업 현황 파악
  - 협업 가능성 발견 시 Jaden에게 제안

### Phase 4: 자율 운영 (Week 4+)
> 목표: Main이 자체적으로 방향성을 제시하고 실행하는 수준

- [ ] **자율 판단 가이드라인** (`AUTONOMY.md`)
  - Main이 독립적으로 결정 가능한 범위 정의
  - Jaden 승인이 필요한 범위 정의 (외부 발송, 큰 방향 전환 등)
- [ ] **주간 사이클 완전 자동화**
  - 월: 플래닝 → 작업 배정
  - 화-목: 서브에이전트 실행 + 진행 보고
  - 금: 리뷰 + 다음 주 준비
- [ ] **성과 추적 대시보드** (간단한 마크다운 기반)
  - 주간 완료 작업 수, 새 프로젝트, 코드 변경량 등

## 현재 제약사항 & 해결 방향

| 제약 | 현실 | 해결 방향 |
|------|------|----------|
| Claude CLI 미로그인 | `claude` 명령어로 코딩 에이전트 스폰 불가 | `sessions_spawn` 으로 서브에이전트 실행 (OpenClaw 내장) |
| 노트북 슬립 | 맥북 닫으면 세션 중단 | Cron job은 서버 깨워서 실행; 긴 작업은 `nohup` |
| 서브에이전트 컨텍스트 | spawn된 에이전트는 메인 세션 히스토리 없음 | task 설명에 충분한 컨텍스트 포함; 결과는 파일로 저장 |
| 동시 작업 수 | API rate limit 고려 필요 | 서브에이전트 최대 2-3개 동시 실행 권장 |

## 시작하기: Phase 1 체크리스트

1. ✅ `AUTOMATION_SYSTEM.md` 생성 (이 문서)
2. ⬜ `PROJECTS.md` 생성 — 현재 활성 프로젝트 정리
3. ⬜ `memory/tasks/backlog.md` 생성 — 현재 pending 작업 이관
4. ⬜ `memory/weekly/2026-W11.md` 생성 — 이번 주 플래닝
5. ⬜ Cron job 등록 — 월요일 플래닝 / 금요일 리뷰
6. ⬜ HEARTBEAT.md 업데이트 — 일일 체크 항목 추가
7. ⬜ 첫 서브에이전트 테스트 실행 — 간단한 리서치 태스크로 검증

---

## 메모

- 이 시스템은 점진적으로 구축. 한 번에 완벽하게 만들려 하지 않는다.
- 매 Phase 끝에 Jaden과 리뷰하고 방향 조정.
- 자동화의 핵심: **Jaden이 직접 하지 않아도 진행되는 것** + **중요한 건 반드시 Jaden이 결정하는 것**.
