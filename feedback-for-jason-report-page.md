## 핵심 방향

격주 레포트의 마케팅 포인트는 **"압도적인 숫자"**입니다.
이 팀이 AI를 극한까지 활용해서 엄청난 양의 코드를 생산하고 있다는 인식을 주는 것이 목표이며, 숫자가 항상 **가장 앞에, 가장 크게, 여러 곳에** 노출되어야 합니다.

---

## 구체적 수정 사항

### 1. 커밋(Commit) 관련 수치 전면 제거
- 커밋 횟수는 더 이상 의미 있는 지표가 아님 (현재 개발 방식상 대규모 코드를 한 번에 커밋하므로 횟수 자체는 임팩트 없음)
- **커밋 관련 표기를 모두 삭제**하고, **코드 라인 변경(Line Changes)** 수치로 대체

### 2. 숫자 표기 방식 변경
- 축약 표기(예: 4.9M) 대신 **풀 숫자로 표기** (예: **4,900,000**)
- 숫자의 임팩트를 최대한 살려야 하며, 축약은 오히려 규모감을 줄임

### 3. 숫자 시각적 강조
- 핵심 숫자(Line Changes 등)의 **폰트 크기를 대폭 키울 것**
- 페이지 상단에서 가장 먼저 눈에 들어오는 요소여야 함
- 현재는 폰트가 작고 시각적 임팩트가 부족

### 4. 상단 통계 항목 변경
기존 항목을 아래와 같이 조정:

| 변경 전 | 변경 후 |
|---------|---------|
| Commits | **삭제** |
| Active Repos | **Active Projects** (명칭 변경) |
| Contributors | **삭제** (외부인 관점에서 조직 규모가 작아 보일 수 있음) |
| Line Changes | **최상단, 최대 크기로 강조** |

→ 최종 상단 표시 항목: **Line Changes (Growth)** + **Active Projects**

### 5. 정렬 기준 변경
- 모든 목록/배열은 **Line Changes 수치 기준 내림차순**으로 정렬
- 카테고리별 정렬, 프로젝트별 정렬 모두 동일 기준 적용
- 알파벳순 정렬 ❌ → 라인 변경량 기준 정렬 ✅

### 6. 랜드스케이프(Landscape) 인포그래픽
- 각 프로젝트 카드에 **코드 변경 수치(+/-)를 반드시 포함**
- 현재는 프로젝트명과 설명만 나열되어 있어 숫자 없이 가시성이 떨어짐
- 프로젝트 설명은 간결하게 유지하되, 수치가 핵심

### 7. 카테고리 구성
- 하나의 카테고리에 **10개 이상의 프로젝트가 모이지 않도록** 세분화
- 필요 시 서브 카테고리 추가 (예: AI → AI Automation / AI + Blockchain 등)
